Operator Guide

Step-by-step operating instructions for running Kinoko ad productions via Claude Desktop + Kie.ai. Read this before starting your first session.

v2.1 · June 2026 Requires Kie.ai MCP

How the Workflow Runs

Every production session follows the same five-stage sequence. Complete each stage fully before moving to the next — skipping steps wastes credits.

1

Open a fresh session and load the master prompt

Start a new Claude Desktop chat. Paste the full contents of KINOKO_AI_GENERATION_MASTER_v2_1.md at the top. Ask Claude to confirm it has read and understood before you continue — this catches MCP connectivity problems early.

2

Prepare your character reference image

Generate the character in Higgsfield (doctor, UGC creator, etc.). Download the image. Upload it to the Kinoko Google Drive folder. Set sharing to "Anyone with the link — Viewer". Copy the share URL and convert it to the direct-download format — see Google Drive URL Rules below.

3

Submit the briefing with all required context

Give Claude the script, the Drive direct-download URL of your reference image, the format (A-Roll or B-Roll), and the target ad length. Example: "Here is the briefing. Use this Drive URL as the reference image for every scene. Use Veo 3 Fast. Follow the master prompt."

4

Always run 3 test clips before the full batch

Ask Claude to generate the first 3 script lines only. Play them back. If the lipsync, voice tone, and face all match expectations, approve and continue. Full batches consume real credits — a failed run costs the same as a successful one.

⚠ Never skip the test phase
5

Generate the full batch, then assemble in your editor

Once the test clips pass, tell Claude to generate all remaining scenes. Download every clip within 24 hours — Kie.ai URLs are temporary and expire. Import into Premiere, CapCut, or DaVinci and cut in script order.

A-Roll vs B-Roll

Every scene in a Kinoko ad is either A-Roll or B-Roll. The distinction determines which tools are called and how the reference image is handled.

A-Roll

Person speaking to camera

Doctor authority spots, UGC creator testimonials, or any scene where a character delivers lines directly on screen.

Video tool Veo 3 Fast — native lipsync in DE / FR / IT
Reference image One Drive image, reused for every line in the batch
B-Roll

Visuals without spoken dialogue

Pixar-style avatars, 3D body visuals, lifestyle shots, product pours, diagrams — anything supporting the script without requiring lipsync.

Video tool Kling — stylised motion, 3–15s clips
Reference image One image per scene, generated fresh via Nano Banana

Generating B-Roll

B-Roll is image-first: Nano Banana generates a still, then Kling animates it. Example sequence for a multi-scene B-Roll batch:

1

Write a description for each B-Roll scene

Note what the scene shows, the mood or energy level, and whether a recurring character appears. Specific descriptions produce consistent output.

2

Paste the master prompt, then the scene descriptions

Always load the master prompt first — it carries the visual standards and tool-routing rules that keep B-Roll on-brand.

3

Tell Claude to use Nano Banana + Kling

Example: "Generate the B-Roll images for these scenes using Nano Banana, then animate each one with Kling."

4

Claude builds a scene map, submits images in parallel, then animates

All image requests are submitted before Kling is called — faster than scene-by-scene. Review Scene 1 first; if the style and character match, approve the rest of the batch.

5

Lock character consistency across scenes

B-Roll images are generated from text, so character drift is possible over a long batch. After Scene 1 is approved, tell Claude to use that first image as the image_input reference for all later scenes.

Essential for Pixar avatar sequences

Iterating When Results Don't Match

If the 3 test clips fall short, do not rerun the whole batch. Target the specific problem and fix only that — then retest before scaling.

Face drifts between scenes

Tell Claude: "Use the approved Scene 1 image as the reference for all remaining scenes." Pinning a single source image locks the face across the batch.

Voice sounds wrong or unnatural

Regenerate the problem clip and add character context to the prompt — for example: "authoritative German doctor, mid-40s, speaking directly to camera." Tone cues significantly improve voice quality.

Audio generates in English instead of German

This is the most common failure. Tell Claude to set enableTranslation: false on every Veo 3 call. Without this flag, Veo 3 may translate the script line before generating audio.

A single scene needs regenerating

Say: "Regenerate Scene [X] — reason: [what's wrong]. Use the same seed and reference image." Individual redos cost approximately $0.30 per clip at Veo 3 Fast rates.

The prompt itself needs revision

Describe the issue in plain terms — "make the character look younger", "add warmer lighting", "remove the mirror from the background." Claude rewrites the prompt and queues another 3-clip test.

Google Drive URL Rules

Kie.ai cannot access images uploaded directly into a Claude chat. Every reference image must be hosted on Google Drive with public sharing enabled, and the URL must be in the direct-download format — not the default share link Google gives you.

✗  Default share link — do not use
https://drive.google.com/file/d/FILE_ID/view?usp=sharing
✓  Direct-download URL — use this
https://drive.google.com/uc?export=download&id=FILE_ID

Copy the FILE_ID from the share link — it is the long alphanumeric string between /d/ and /view — then paste it into the direct-download pattern above.

Critical — Sharing permission

Always set the Drive file to "Anyone with the link — Viewer" before copying the URL. If sharing is restricted, Kie.ai fails silently — the batch appears to run but produces no output, and the credits are still consumed.

Which Drive folder to use

Always upload to the Kinoko-owned shared Drive folder, not a personal Google account. Personal Drive files can have quota and access limits that cause intermittent failures on large batches, even when set to "Anyone with the link."

Kie.ai MCP Setup

One-time setup per machine. The MCP server lets Claude Desktop call Kie.ai tools directly during a chat. If anything goes wrong after setup, share the error message with Claude in chat — it can usually diagnose the problem.

Before You Start — Both Platforms

  • Install Claude Desktop from claude.ai/download
  • Log in to Kie.ai and generate an API key from your account settings
  • Install Node.js LTS from nodejs.org — the MCP server package requires it
  • Store your Kie.ai API key in a password manager — you will paste it into a config file

Platform Setup Steps

  • Close Claude Desktop if it is currently running
  • Open File Explorer and paste %APPDATA%\Claude into the address bar
  • Open claude_desktop_config.json in Notepad — or create it if it does not exist
  • Paste the config snippet below and replace YOUR_KIE_AI_KEY with your actual key
  • Save the file, then restart Claude Desktop
  • Open a new chat and type "list available tools" — you should see nano_banana_image, veo3_generate_video, and kling_video. If not, ask Claude to help debug.
  • Close Claude Desktop if it is currently running
  • Open Finder → press ⌘ + Shift + G → paste ~/Library/Application Support/Claude
  • Open claude_desktop_config.json in TextEdit — or create it if it does not exist
  • Paste the config snippet below and replace YOUR_KIE_AI_KEY with your actual key
  • Save as plain text and restart Claude Desktop
  • Open a new chat and type "list available tools" — confirm the Kie.ai tools are listed. If not, ask Claude to diagnose.

Config Snippet — Both Platforms

{ "mcpServers": { "kie-ai": { "command": "npx", "args": ["-y", "@kie-ai/mcp-server"], "env": { "KIE_AI_API_KEY": "YOUR_KIE_AI_KEY" } } } }

The package name @kie-ai/mcp-server may change. If the connection fails, check the Kie.ai documentation for the current package name and update the args field.

Stuck on something?

Ask Claude directly in chat — it can diagnose MCP connection issues, walk through config errors, and suggest prompt fixes. Share the exact error message and what you already tried.

Common Mistakes to Avoid

Uploading reference images directly into the Claude chat

Kie.ai has no access to files attached in Claude's chat window. Always upload to Google Drive first, set sharing to "Anyone with the link — Viewer", and paste the direct-download URL into the session.

Running the full batch before testing

Always generate 3 clips first, verify quality, then scale. A full batch that fails due to a wrong URL or incorrect language setting loses real money with nothing to show for it.

Not downloading clips within 24 hours

Kie.ai output URLs expire after approximately 24 hours. Download every clip on the same day you generate it — the session log does not preserve access to expired files.

Using a personal Google Drive instead of the Kinoko folder

Always upload to the Kinoko-owned shared Drive. Personal Drive links can fail when fetched programmatically even when they appear accessible in a browser.

Starting a new session without reloading the master prompt

Claude has no memory between separate chat sessions. Every new session starts blank. Paste the master prompt and any active briefing context at the top of each new chat before doing anything else.

Higgsfield still works for manual generation

The Claude + Kie.ai workflow is faster and cheaper for batches, but nothing forces you to use it exclusively. Higgsfield remains available for one-off manual generations. The speed and cost advantages of the Claude workflow are most visible on full ad batches — try it on your next job.
